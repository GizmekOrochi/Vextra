Detailed Description

Global constants and configuration for the Vextra CLI framework.

This header centralizes all application-wide settings, constants, and default template values used throughout the Vextra project.

Purpose

  • Define metadata such as app name and version.
  • Store the default directory structure for generated projects.
  • Provide visual icons for standardized terminal output.
  • Ensure consistent values across all commands.

Design philosophy

This file acts as a single source of truth for configuration. No hardcoded values should appear directly in command implementations.

Example:

std::cout << vextra::ICON_SUCCESS << " Project created.\n";
std::filesystem::create_directories(vextra::SRC_DIR);
